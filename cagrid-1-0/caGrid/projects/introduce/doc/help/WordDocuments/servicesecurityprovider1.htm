Service Security Provider

The Introduce Service Security API is utilized by every Introduce service. The wsdl file of every service will have the operation getServiceSecurityMetadata() imported and will use the ServiceSecurityProvider implementation of that method. This implementation will enable the service to provide the client with a description of the security requirements needed for the client to invoke any particular operation on the service. Without this type of information a client would have to go by trial and error to determine what the service required for communicating with it. The getServiceSecurityMetadata() operation will have not security requirements so that anyone can contact it and retrieve the information. Introduce generated clients will automatically do this and will use the information to automatically configure the stub with the appropriate security settings required to make the invocation.

Authorization

Introduce uses the Globus PDP infrastructure for authorization. The PDP framework enables a class or series of classes which implement the PDP or PIP interfaces to be chained together. This chain of authorization classes can be added at the service, method, or resource level. These classes will loaded and called serialy each time the service is invoked and the service descriptor indecates the chain needs to be used. The classes will each be able to accept or reject the call to the method based on the credientials of the caller or any other information they wish to recieve from any place else. Introduce will automaticaly generate the PDP class to support authorization based on GridGrouper and/or CSM. This class will automatically be generated by the Synchronization Engine and written to <service package name>.service.globus.Authorization.java. The user also has the ability to put in a custom PDP chain. The Introduce enables this and when this is the case Introduce will write there specified PDP chain string into the security-descriptor.xml for the servce instead of the one that Introduce generates. For more detail on the service side security configuration of a globus service please see the Globus documentation.
